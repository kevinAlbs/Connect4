Connect Four API Endpoint

The source is on the GitHub repository

There is currently one functional endpoint (more to come). All can be used as either GET or POST requests.

http://kevinalbs.com/connect4/back-end/index.php/getMoves

Main Checking

Parameters

  • board_data A single string containing 49 characters, all mapping to locations on the 7 x 7 connect four board (going left to right, top to bottom)
  • player (optional) Which player (1 or 2) to find moves for. Default is player 1.

This returns a JSON object. It contains the column of the move (zero-based, so 0 is the left-most column and 6 is the right-most) mapping to it's relative min-max score

Example curl request:

curl 'http://kevinalbs.com/connect4/back-end/index.php/getMoves?board_data=0000000000000000020000001200000210000021001012100&player=2'
http://kevinalbs.com/connect4/back-end/index.php/hasWon

Checks if a board is a winning board for a player. Only checks for a single piece.

  • board_data (same as above) A single string containing 49 characters, all mapping to locations on the 7 x 7 connect four board (going left to right, top to bottom)
  • player (optional) Which player (1 or 2) to find moves for. Default is player 1.
  • i The ith row (with 0 being top and 6 being bottom) of the piece to check
  • j The jth column (with 0 being left and 6 being right) of the piece to check

Example curl request:

curl 'http://localhost/connect4/back-end/index.php/hasWon?board_data=0002000001102000222100012220002111000122100012110&player=2&i=3&j=3'
